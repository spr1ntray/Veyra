# Grimoire Autocast — Детальный GDD

> **Платформа**: ПК браузер (HTML5/CSS/Vanilla JS)
> **Жанр**: Idle/RPG, dark fantasy
> **Ключевое решение**: ВСЕ стратегические решения принимаются ДО боя. Бой полностью автоматический.
> **Философия**: Прокачка и подготовка решают. Скилл игрока = 0. Мозг игрока = выбор заклинаний под конкретного врага.

## 1. UI гримуара (экран подготовки к бою)

### Общее описание

Экран подготовки появляется между выбором врага и началом боя. Занимает весь экран. В центре — раскрытый гримуар (книга) с двумя страницами:

- **Левая страница**: 5 слотов для заклинаний (горизонтальная полоска слотов, пронумерованных I-V)
- **Правая страница**: пул доступных заклинаний (все открытые заклинания игрока, сгруппированные по школам)

Под гримуаром — карточка врага: иконка, имя, HP, известные резисты/слабости.

### Взаимодействие

| Действие | Как работает |
|----------|-------------|
| Назначить заклинание в слот | Клик по заклинанию в пуле, затем клик по слоту. Или: drag & drop из пула в слот |
| Убрать заклинание из слота | Клик по занятому слоту (заклинание возвращается в пул) |
| Поменять порядок | Drag & drop между слотами |
| Дублирование | Одно заклинание можно поставить в несколько слотов |
| Быстрый набор | Двойной клик по заклинанию — встает в первый свободный слот |
| Горячие клавиши | 1-5 = выбрать слот, затем клик по заклинанию; или перетаскивание |

### Правила сборки

1. Количество слотов: **5** (фиксировано, не меняется с уровнем)
2. Минимум для старта боя: **3 заполненных слота** (слоты IV-V могут быть пустыми; пустые слоты пропускаются в ротации)
3. Дублирование разрешено: можно поставить Arcane Bolt во все 5 слотов
4. Нет ограничений на школы: можно миксовать любые школы
5. Порядок имеет значение: маг кастует строго I -> II -> III -> IV -> V -> I -> ...

### Превью DPS

На левой странице в реальном времени отображается расчёт:

- **Est. DPS**: средний урон в секунду с учётом статов игрока и резистов выбранного врага
- **Est. Kill Time**: ожидаемое время убийства (HP врага / DPS)
- Обновляется мгновенно при каждом изменении слотов

Формула превью: `DPS = SUM(avg_dmg_i * school_modifier_vs_enemy) / SUM(cast_time_i)` для всех заполненных слотов.

### Handoff: UI/UX Designer

- Визуальный макет раскрытого гримуара: пожелтевшие страницы, рунные узоры на краях, dark fantasy стиль
- Анимация: заклинание "впечатывается" в слот при назначении (чернила проступают на странице)
- Анимация drag: заклинание парит за курсором как светящийся символ
- Hover на заклинании в пуле: тултип с полной статистикой

## 3. Влияние прокачки (формулы урона)

### Характеристики персонажа

Текущая формула из `state.js`:

```
Intelligence = 5 + (level - 1) * 3 + BonusPower
BonusPower = SUM(equipment.bonus)  // из всех 3 слотов экипировки
```

Примеры:

| Уровень | Экипировка | BonusPower | Intelligence |
|---------|-----------|------------|--------------|
| 1 | Starter (0+0+0) | 0 | 5 |
| 3 | Common (3+3+3) | 9 | 20 |
| 5 | Uncommon (12+14+15) | 41 | 58 |
| 8 | Rare (20+22+24) | 66 | 87 |
| 10 | Epic (30+35+38) | 103 | 135 |

### Формула урона заклинания

```
final_damage = floor(base_roll * INT_multiplier * school_modifier * debuff_modifier * focus_modifier * buff_modifier)
```

Где:

| Компонент | Формула | Пример |
|-----------|---------|--------|
| **base_roll** | randInt(spell.minDmg, spell.maxDmg) | 25-40 для Arcane Bolt |
| **INT_multiplier** | 1 + (Intelligence - 5) / 100 | INT 50 -> x1.45 |
| **school_modifier** | Зависит от врага (см. раздел 5) | x1.3 (слабость) или x0.7 (резист) |
| **debuff_modifier** | 1.0 базово, 1.15 если Void Eruption активен на враге | x1.15 |
| **focus_modifier** | 1.0 базово, 2.0 если предыдущий каст был Focus | x2.0 |
| **buff_modifier** | Mana Surge: x1.25. Множители перемножаются. | x1.25 |

### Формула щита (Mana Shield)

```
shield_HP = 40 + floor(Intelligence * 0.8)
```

| Intelligence | Щит HP |
|-------------|--------|
| 5 | 44 |
| 20 | 56 |
| 58 | 86 |
| 135 | 148 |

### Формула лайфстила (Shadow Bolt, Drain Life)

```
heal = floor(final_damage * lifesteal_percent)
```

- Shadow Bolt: lifesteal_percent = 0.20
- Drain Life: lifesteal_percent = 0.50

Хил не может превысить недостающее HP мага (не дает оверхил).

### Формула DoT (Ignite)

```
dot_tick_damage = floor(8 * INT_multiplier * school_modifier)
```

- 3 тика с интервалом 1.5 сек
- Стакается до 3 раз (каждый стак — отдельный таймер)
- Максимум DoT при 3 стаках: 9 тиков по 8 * INT_multiplier

### Влияние конкретных предметов

Текущая система предметов дает только +BonusPower. Это сохраняется без изменений — все предметы усиливают INT, а INT усиливает все заклинания одинаково.

**Будущее расширение (пометка для следующей итерации)**: предметы с бонусами к конкретным школам (+Fire damage %, +Shadow lifesteal %). Не делать в MVP.

## 4. Механика боя (автокаст)

### Общий flow

```
[Grimoire Setup] -> [Begin Battle] -> [Auto-Combat Loop] -> [Battle End] -> [Rewards]
                                            |
                                            v
                                    Маг кастует заклинания
                                    из слотов I->II->...->V->I->...
                                    пока враг жив или маг мёртв
                                    или истёк таймер
```

### Параметры боя

| Параметр | Значение |
|----------|----------|
| Максимальное HP мага | 100 + (level - 1) * 15 + floor(BonusPower * 0.5) |
| Таймер боя | 60 секунд |
| Пауза между кастами | 0.3 сек (после окончания cast_time текущего заклинания) |
| Пауза перед первым кастом | 1.0 сек (вступительная анимация) |

HP мага по уровням:

| Уровень | BonusPower (avg) | Max HP мага |
|---------|-----------------|-------------|
| 1 | 0 | 100 |
| 3 | 9 | 134 |
| 5 | 41 | 180 |
| 8 | 66 | 238 |
| 10 | 103 | 286 |

### Скорость кастов и ротация

Маг кастует заклинания строго по порядку слотов. Каждое заклинание имеет свой cast_time. Между кастами — фиксированная пауза 0.3 сек.

Пример ротации: Focus (1.0s) -> Fireball (2.2s) -> Arcane Bolt (1.8s) -> Fireball (2.2s) -> Shadow Bolt (1.8s)

Полный цикл: (1.0 + 0.3) + (2.2 + 0.3) + (1.8 + 0.3) + (2.2 + 0.3) + (1.8 + 0.3) = 10.5 сек на один полный проход гримуара.

### Действия врага

Враг атакует автоматически с фиксированным интервалом (attack_speed). Каждая атака наносит фиксированный урон. Урон врага уменьшается щитом (Mana Shield), замедление (Frost) увеличивает интервал атаки врага.

```
enemy_attack_interval = base_attack_speed / (1 - slow_percent)
```

Пример: враг с attack_speed 2.0 сек под Blizzard (40% slow):
```
interval = 2.0 / (1 - 0.4) = 3.33 сек
```

### Grimoire Tracker

Полоска из 5 слотов внизу боевого экрана. Показывает полную ротацию. Текущее кастуемое заклинание подсвечено и пульсирует. Следующее — слегка подсвечено. Остальные — тусклые. Когда каст завершен, индикатор сдвигается к следующему слоту.

### Визуальное отображение каста

1. **Cast start** (0 мс): маг поднимает посох, слот в Tracker начинает пульсировать
2. **Projectile launch** (50% cast_time): снаряд вылетает от мага к врагу, цвет = школа
3. **Impact** (100% cast_time): враг получает урон, число урона всплывает
4. **Cooldown** (cast_time + 300мс): пауза, индикатор переключается на следующий слот

Для Focus: нет снаряда, маг окутывается золотым свечением, индикатор "FOCUS x2" появляется над магом.

Для Mana Shield: нет снаряда, голубой щит появляется вокруг мага, число щита всплывает.

Для DoT (Ignite): при ударе иконка огня появляется над врагом, тикает визуально каждые 1.5 сек.

### Условия окончания боя

| Условие | Результат | Награда |
|---------|-----------|---------|
| HP врага <= 0 | Победа | Полная награда |
| HP мага <= 0 | Поражение | 50% золота, 50% XP, нет дропа |
| Таймер = 0, враг жив | Ничья (Timeout) | 30% золота, 30% XP, нет дропа |

### Edge Cases

- **Focus + Focus подряд**: второй Focus не перезаписывает первый; он тратит слот впустую (лог: "Focus already active, wasted"). Намеренно — игрок должен думать о расстановке.
- **Mana Shield + Mana Shield подряд**: второй перезаписывает первый (новый щит). Не стакается.
- **DoT + смерть врага**: если DoT-тик убивает врага — считается победой, бой заканчивается.
- **Щит + лайфстил**: лайфстил считается от полного урона, не от урона после щита врага. Хил идёт в HP мага, не в щит.
- **Пустой слот в ротации**: пропускается, следующий каст идёт без паузы.
- **Все 5 слотов = Focus**: маг ничего не кастует (Focus не атакует), таймер истечёт. Timeout. Валидация при сборке гримуара: предупреждение "No damage spells in grimoire!"
- **Замедление + DoT одновременно на враге**: оба работают независимо.

### Модификатор школы урона (school_modifier)

Если школа заклинания совпадает со слабостью врага — урон умножается на значение в таблице (x1.3, x1.5). Если совпадает с резистом — урон умножается на значение резиста (x0.5, x0.7). Arcane — нейтральная школа, по умолчанию x1.0 ко всем врагам (если не указано иное). Utility-заклинания (Shield, Focus) не наносят урон — school_modifier не применяется.

### Как игрок узнает слабости/резисты ДО боя

**Система "Бестиарий" (Bestiary)**:

1. **Первая встреча**: враг показан с иконкой "?" на резистах/слабостях. Игрок идёт в бой вслепую.
2. **После первого боя** (победа или поражение): тип врага "изучен". Резисты и слабости полностью раскрываются в бестиарии и на карточке врага в экране подготовки гримуара.
3. **Визуальные подсказки**: цвет ауры врага на спрайте намекает на стихию (красная аура = fire resist, синяя = frost resist). Внимательный игрок может догадаться до первого боя.

Бестиарий доступен через кнопку в HUD. Показывает всех встреченных врагов и их полные характеристики.

### Выбор врага

На экране локации (Town Square или другая локация) — список доступных врагов для боя. Враги разблокируются по уровню игрока:

## 6. Прогрессия

### XP и награды за бой

### Ежедневные бои

Лимит: **5 боёв в день** (сохраняется из текущей системы, сейчас закомментировано для тестов). Training Dummy не тратит слот боя.

Бонус за 5 побед подряд за день: +25 золота, +50 XP (сохраняется из текущей системы).

## 7. Что менять в существующем коде

### combat.js — что УБРАТЬ

| Что | Почему |
|-----|--------|
| Ручной вызов `castSpell(spellId)` через клик по кнопке | Бой автоматический, игрок не кликает кнопки заклинаний |
| `ACTIONS_TOTAL = 10` (лимит действий) | Заменяется таймером 60 сек + HP мага |
| `DUMMY_MAX_HP = 200` (единственный враг) | Заменяется системой врагов с разным HP |
| `battleState.focusCharged` (ручное отслеживание Focus) | Focus остается, но логика переносится в автокаст-цикл |
| `initSpellButtons()` в main.js | Кнопки заклинаний не нужны на боевом экране |
| `setSpellButtonsDisabled()` | Нет кнопок — нечего блокировать |

### combat.js — что СОХРАНИТЬ

| Что | Почему |
|-----|--------|
| `calcDamage()` — формула урона | Базовая формула остается, расширяется school_modifier |
| `playSpellAnimation()` — визуал снаряда | Адаптировать под разные школы/цвета, но структура та же |
| `showDamageNumber()` — всплывающие цифры | Сохранить как есть |
| `updateDummyHP()` -> переименовать в updateEnemyHP() | Логика HP-бара остается |
| `addCombatLog()` — лог боя | Расширить для новых эффектов (DoT, heal, shield) |
| `endBattle()` — завершение боя и награды | Расширить для новых типов наград по врагам |
| `delay()` — утилита | Без изменений |

### combat.js — что ДОБАВИТЬ

| Что | Описание |
|-----|----------|
| `ENEMIES` объект | Данные всех врагов (HP, атака, скорость, резисты, слабости) |
| `SPELLS` расширить | 12 заклинаний вместо 2. Добавить поля: school, castTime, effect, effectParams |
| `grimoireSlots[5]` | Массив ID заклинаний в слотах I-V |
| `runAutoCombat()` | Главный цикл: async функция с setTimeout/setInterval. Итерирует по grimoireSlots, вызывает castSpell для каждого, ждёт castTime + 0.3s, переходит к следующему. Параллельно: враг атакует по своему таймеру. |
| `enemyAttackLoop()` | Отдельный setInterval для атак врага. Проверяет щит, наносит урон магу, проверяет смерть мага. |
| `applyDoT()`, `tickDoT()` | Логика периодического урона |
| `applySlowDebuff()` | Изменение attack_speed врага |
| `applyVulnDebuff()` | Void Eruption: +15% incoming damage |

## Приложение A: Пример полного боя (simulation)

**Игрок**: уровень 5, INT = 58 (Uncommon шмотки)
**Враг**: Undead Knight (HP 450, ATK 15, speed 2.0s, Resist Shadow x0.7, Weak Fire x1.3)
**Гримуар**: [Focus] [Fireball] [Ignite] [Fireball] [Frost Spike]

```
INT_multiplier = 1 + (58-5)/100 = 1.53

0.0s  — Бой начинается. Маг HP: 180/180
1.0s  — Cast: Focus (cast_time 1.0s) — Buff "next spell x2"

         Враг HP: 450 - 167 = 283
3.8s  — Пауза
4.0s  — Враг атакует: -15 HP. Маг: 150/180
5.3s  — Cast: Ignite (1.5s). base_roll = 12.
         damage = floor(12 * 1.53 * 1.3) = floor(23.9) = 23
         DoT applied: 8 * 1.53 * 1.3 = 15.9 -> 15 per tick, 3 ticks
         Враг HP: 283 - 23 = 260
5.6s  — Пауза
6.0s  — Враг атакует: -15 HP. Маг: 135/180
6.8s  — DoT tick #1: -15. Враг HP: 260 - 15 = 245
7.8s  — Cast: Fireball (2.2s). base_roll = 38.
         damage = floor(38 * 1.53 * 1.3) = floor(75.6) = 75
         Враг HP: 245 - 75 = 170
8.0s  — Враг атакует: -15 HP. Маг: 120/180
8.1s  — Пауза
8.3s  — DoT tick #2: -15. Враг HP: 170 - 15 = 155
9.6s  — Cast: Frost Spike (1.5s). base_roll = 25.
         school = Frost, нейтральная к Undead Knight (x1.0)
```

## Приложение B: Стратегические решения игрока

Хотя бой автоматический, стратегическая глубина — в подготовке:

| Решение | Trade-off |
|---------|-----------|
| Больше Focus в ротации | Выше пик-урон, но меньше общий DPS (Focus не делает урон сам) |
| Все Fire vs Undead Knight | Максимальный урон, но нет контроля/хила — можно умереть |
| Drain Life + Shadow Bolt vs Demon Lord | Demon Lord резистит Fire, и бьёт сильно. Shadow lifesteal = выживание. Но Shadow нейтральна к нему, бой дольше. |
| Mana Shield + Frost + нюки | Щит поглощает урон, замедление снижает DPS врага. Survivability сборка. |
| Ignite spam (3x Ignite) | 3 стака DoT = 45/тик. Огромный sustained damage. Но прямой урон низкий, бой долгий, можно не уложиться в таймер vs танки. |
| 1 Void Eruption + 4 нюка | Vuln дебафф (+15%) усиливает 4 следующих каста. Оптимально для длинных боёв. |
