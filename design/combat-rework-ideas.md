# Переработка боевой системы Veyra -- 5 идей

> **Контекст**: браузерная мобильная idle/RPG, dark fantasy, чистый HTML5/CSS/Vanilla JS. Текущая система -- маг бьёт манекен Arcane Bolt / Focus, 10 действий, 5 боёв в день. Проблемы: однообразно, визуально скучно, нет тактической глубины.

---

## Идея 1: Руническая цепочка (Rune Chain)

### Описание механики
На экране появляется горизонтальная полоса с 5 ячейками. Каждый ход игрок выбирает одну из 3-4 рун (элементы: огонь, тень, аркана, защита) и ставит в следующую ячейку. Когда ячейка заполняется -- руна автоматически кастуется на врага. Ключ: соседние руны дают combo-бонусы. Огонь + Огонь = удвоенный урон. Тень + Аркана = пробитие брони. Три одинаковых подряд = ультимативное заклинание с особой анимацией. Враг имеет HP-бар и свой паттерн атак (предсказуемый через иконки-подсказки).

### Что делает игрок руками
- Тапает по одной из 3-4 рун внизу экрана (большие кнопки, удобные для пальца)
- Видит preview следующей атаки врага (иконка над врагом) и решает: атаковать или поставить защитную руну
- Выбор стратегический: собрать combo из 3 одинаковых рун или миксовать для пробитий

### Визуальный стиль боя
- Руны светятся цветами элементов в ячейках (amber, purple, blue, green)
- Линия из заполненных ячеек пульсирует как магическая цепь
- При combo 3-в-ряд -- экран вспыхивает цветом элемента, крупная анимация удара
- Враг реагирует: откидывается, трескается щит, горит
- Фон арены: тёмный каменный зал с факелами, дым от заклинаний оседает

### Плюсы
- Тактическая глубина при минимуме действий (1 тап в ход)
- Combo-система даёт "aha!"-моменты и реиграбельность
- Прост в реализации: 5 div-ячеек + CSS-анимации
- Хорошо масштабируется: новые руны = новые combo

### Минусы
- Нужен AI-паттерн для врагов (хотя бы таблица фиксированных ходов)
- Может казаться слишком простым после 50+ боёв без новых рун
- Combo-таблицу надо балансировать аккуратно

### Оценка сложности реализации
Средняя. Основной объём -- combo-таблица и анимации переходов между рунами.

## Идея 2: Ритуальные символы (Ritual Glyphs)

### Описание механики
Бой полу-idle: маг автоматически атакует каждые 2 секунды базовым заклинанием. Игрок усиливает атаки, рисуя пальцем простые символы (свайп вверх, круг, зигзаг, тап-тап) прямо на боевом экране. Каждый символ -- усиление следующей атаки определённым элементом. Точность и скорость рисования влияют на множитель урона (x1.0 при плохом -- x2.5 при идеальном). Если игрок ничего не делает -- маг всё равно бьёт, но слабо.

### Что делает игрок руками
- Свайпы и жесты по экрану в момент между автоатаками (окно 2 секунды)
- Выбирает какой символ рисовать в зависимости от слабости врага (иконка-подсказка)
- Может ничего не делать -- idle-режим, маг бьёт базовый урон

### Визуальный стиль боя
- На экране остаётся затухающий след пальца -- светящийся как магические чернила
- При успешном распознавании символ вспыхивает и "влетает" во врага
- Автоатака -- тусклый серый снаряд; усиленная -- яркий цветной с шлейфом частиц
- Таймер между атаками -- круговой прогресс-бар вокруг аватара мага
- Фон: руинированная башня, тлеющие руны на стенах

### Плюсы
- Настоящий idle: можно вообще не трогать экран -- бой продолжится
- Жестовый ввод уникален для мобильного -- приятная тактильность
- Высокий skill ceiling: опытный игрок наносит x2.5, новичок x1.0
- Визуально впечатляет: светящиеся следы на тёмном фоне

### Минусы
- Распознавание жестов на Canvas/Touch API -- самая сложная часть кодинга
- На маленьких экранах жесты могут конфликтовать со скроллом
- Нужна калибровка: что считать "хорошим" свайпом, а что мусором
- Accessibility: недоступно для игроков с ограниченной моторикой (нужен fallback на тапы)

### Оценка сложности реализации
Высокая. Gesture recognition -- нетривиальная задача, даже для простых символов.

## Идея 3: Карточный залп (Spell Cards)

### Описание механики
Перед боем игрок формирует "колоду" из 5-7 карт заклинаний (из доступных). В бою каждый ход открывается "рука" из 3 случайных карт из колоды. Игрок выбирает одну карту -- она применяется. Бой длится 6-8 ходов. Карты имеют: урон, ману, тип (атака/защита/дебафф). Мана -- общий ресурс: 10 единиц на бой, каждая карта стоит 1-4 маны. Дешёвые карты -- слабые; дорогие -- мощные. Стратегия: когда потратить ману на большой удар, а когда экономить.

### Что делает игрок руками
- Тапает по одной из 3 карт на руке каждый ход
- Перед боем: drag & drop карт в колоду на экране подготовки (одноразовая настройка)
- Может тапнуть по карте для просмотра деталей (hold)

### Визуальный стиль боя
- 3 карты внизу экрана: тёмные рамки с золотым тиснением, иконки элементов
- Выбранная карта "вылетает" вверх, раскрывается, эффект применяется к врагу
- Неиспользованные карты уходят вниз (discard) с fade-анимацией
- Мана-бар: ряд мерцающих точек под картами
- При low mana (<= 2) карты с высокой стоимостью затемняются и трясутся

### Плюсы
- Deckbuilding + roguelike элемент -- огромная реиграбельность
- 3 карты на выбор = осмысленное решение каждый ход
- Хорошо интегрируется с прогрессией: новые карты как награды за уровни/дропы
- Визуально элегантно: карты -- привычная и понятная метафора

### Минусы
- Нужна система карт поверх текущей системы предметов (параллельная прогрессия)
- RNG рандом руки может фрустрировать ("вытянул 3 слабых карты подряд")
- Drag & drop колоды сложнее на тачскрине чем кажется
- Больше контента: нужно 15-20 уникальных карт для минимальной глубины

### Оценка сложности реализации
Средне-высокая. Основной объём -- система колоды, рандом руки, экран конструктора колоды.

## Идея 4: Временная линия (Timeline Duel)

### Описание механики
Бой представлен как горизонтальная временная шкала (таймлайн). Маг и враг -- два токена на этой шкале. Каждое заклинание имеет cast time (0.5-3 секунды реального времени) -- чем мощнее, тем дольше каст. Игрок видит, что враг начал кастовать (иконка заклинания над токеном врага + прогресс-бар). Задача: выбрать заклинание, которое успеет сработать до удара врага, или прервать каст врага дешёвым контрзаклинанием. Бой длится 30-45 секунд реального времени.

### Что делает игрок руками
- Тапает по одному из 3-4 заклинаний в панели внизу
- Наблюдает за прогресс-барами каста мага и врага -- timing решает
- Решает: дождаться окончания каста мощного заклинания или прервать и кинуть быстрое
- Может тапнуть на каст врага чтобы увидеть описание (что будет, сколько урона)

### Визуальный стиль боя
- Таймлайн в центре: тёмная полоса с тикающими делениями, золотые маркеры
- Токен мага (слева) и врага (справа) пульсируют во время каста
- Прогресс-бар каста: круговой, заполняется с tick-эффектом; при завершении -- вспышка
- Быстрые заклинания: острый визуальный удар, молния; медленные: нарастающее сияние, взрыв
- Интеррапт: красная вспышка, иконка заклинания врага трескается и рассыпается

### Плюсы
- Ощущение дуэли в реальном времени -- высокое напряжение
- Простой ввод (тапы) с глубоким тактическим слоем (timing, приоритизация)
- 30-45 секунд -- идеальная длина для мобильной сессии
- Визуально динамично: постоянное движение на экране

### Минусы
- Real-time на мобильном -- нужна отзывчивость <100ms, setTimeout может лагать
- Сложнее балансировать чем пошаговую систему
- Не совсем idle: нельзя отвлечься, каждая секунда важна (нет "пассивного" режима)
- Нужна система интеррапта с граничными случаями (два каста завершаются одновременно)

### Оценка сложности реализации
Средняя. Технически -- таймеры и прогресс-бары на CSS transitions + requestAnimationFrame. Главная сложность -- баланс cast times.

## Идея 5: Элементальная рулетка (Arcane Slots)

### Описание механики
Комбинация idle + active. На экране 3 слота (как в слот-машине), каждый крутится показывая элементы (огонь, лёд, тень, аркана, щит). Слоты останавливаются автоматически слева направо с интервалом 1 секунда. Игрок может тапнуть по слоту чтобы "зафиксировать" его на нужном элементе в момент когда видит подходящий (skill-based timing). Комбинация 3 элементов определяет заклинание: 3 огня = Inferno (макс урон), огонь+аркана+щит = Fire Shield (урон + блок), 3 щита = полный блок вражеской атаки. Враг атакует между спинами. Бой: 5-7 спинов.

### Что делает игрок руками
- Тапает по слоту чтобы остановить его на нужном символе (timing-based skill)
- Может не тапать вообще -- слоты остановятся сами (idle-режим, результат случайный)
- Между спинами видит что враг готовит (подсказка) и решает: нужен щит или урон

### Визуальный стиль боя
- 3 вертикальных барабана с руническими символами, золотые рамки, тёмный фон
- Барабаны крутятся с motion blur, при остановке -- вспышка + click-звук (визуальный)
- Совпадение 3-в-ряд: экран заливает цветом элемента, крупная анимация заклинания
- Частичное совпадение (2 из 3): менее яркий эффект, среднее заклинание
- Промах (все разные): тусклый серый удар, маг качает головой

### Плюсы
- Идеальный idle/active баланс: можно играть пассивно или активно
- Gambling-психология: "ещё один спин" -- высокая retention
- Визуально яркий, игроки узнают паттерн слот-машины мгновенно
- Простая реализация: CSS-анимация вращения + timing check по offsetTop

### Минусы
- Может ассоциироваться с гэмблингом (этический риск для некоторых рынков)
- Skill ceiling ниже чем у вариантов 1, 3, 4 -- timing единственный навык
- Комбо-таблица из 4 элементов x 3 слота = много комбинаций для балансировки
- При idle-режиме полностью случайный результат может фрустрировать

### Оценка сложности реализации
Низкая-средняя. CSS-анимация вращения + tapEvent + lookup-таблица комбинаций.

## Рекомендация

Для Veyra с учётом idle-ядра, мобильного фокуса и dark fantasy стиля:

**Первый приоритет: Идея 5 (Arcane Slots)** -- минимальная сложность реализации, отличный idle/active баланс, визуально впечатляет, легко интегрируется с текущей системой заклинаний и предметов. Элементальные руны вместо фруктов -- естественная dark fantasy обёртка.

**Второй приоритет: Идея 1 (Rune Chain)** -- если нужна большая тактическая глубина. Combo-система хорошо масштабируется с прогрессией и новыми рунами как дропом.

**Не рекомендую для MVP: Идея 2 (Ritual Glyphs)** -- gesture recognition слишком рискованна по срокам и багоёмкости для Vanilla JS без библиотек.

## Зависимости (для всех вариантов)

| Система | Как затрагивается |
|---------|-------------------|
| `state.js` -- combat | Новые поля: текущий враг, его HP/паттерн, результат спина/руны |
| `state.js` -- equipment/buffs | Бонусы от предметов влияют на базовый урон / множители |
| `main.js` -- onBattleEnd | Расширить результат боя новыми данными (combo count, timing score) |
| `combat.css` | Полная переработка визуала арены под выбранную механику |
| `index.html` | Новая HTML-структура боевого экрана |

### Handoff для Coder
- Выбранная механика должна быть полностью описана с numeric parameters перед передачей в разработку
- Нужен отдельный документ `/design/combat-balance.md` с таблицами урона, HP врагов, combo-множителей
- Анимации описать покадрово (timing, easing, CSS properties) в `/design/combat-animations.md`

### Handoff для UI/UX Designer
- Wireframe боевого экрана под мобильное разрешение 375x667 (iPhone SE) и 390x844 (iPhone 14)
- Hit-зоны кнопок: минимум 48x48px для touch targets
- Состояния: idle, casting, hit, victory, defeat
