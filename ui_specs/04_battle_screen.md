# BattleScreen — Тренировочная площадка (Бой)

## Описание
Экран боя с манекеном. Вид от третьего лица: маг на переднем плане спиной к игроку, манекен на дистанции. Игрок выбирает одно из трёх заклинаний за раунд. Три раунда — один бой.

---

## Допущения
- [ASSUMPTION] Фон экрана — статичный арт тренировочной площадки (каменный дворик, ночь, факелы).
- [ASSUMPTION] Маг и манекен — CSS/SVG или спрайт-анимации, не 3D.
- [ASSUMPTION] Выбор заклинания возможен только один раз за раунд — после клика кнопки заклинания становятся disabled до завершения анимации атаки.
- [ASSUMPTION] Манекен не атакует игрока (нет HP у игрока в MVP).
- [ASSUMPTION] Цифра урона появляется над манекеном как floating number.

## Состояния кнопок заклинаний

### Default
- Bg: соответствующий тёмный фон (синий/зелёный/фиолетовый).
- Border: 1px цвет заклинания, opacity 0.6.
- Text opacity: 1.0.
- Cursor: pointer.

### Hover
- Border opacity: 1.0.
- Box-shadow снаружи: `0 0 16px [цвет заклинания]66`.
- Bg: немного светлее (прибавить ~10% яркости).
- Scale: 1.02.
- Transition: 0.15s.

### Active (при клике)
- Scale: 0.97.
- Border: 2px.
- Transition: 0.1s.

### Disabled (пока идёт анимация атаки)
- Opacity: 0.4.
- Cursor: not-allowed.
- Нет hover-эффекта.
- Transition: opacity 0.2s при переходе в disabled.

### Disabled-permanent (бои исчерпаны — не используется на этом экране, бои исчерпаны = экран недоступен)
- Не применимо: экран боя не открывается при fightsToday === 5.

## Анимации атаки

### Последовательность после клика на заклинание

1. **t=0ms**: Все кнопки → disabled (opacity 0.4).
2. **t=0–400ms**: Анимация каста мага:
   - `mage_figure` покачивается: translateX -6px → +6px → 0, 400ms.
   - Над посохом мага появляется цветной свет (box-shadow 30px цвет заклинания).
3. **t=400–800ms**: Полёт снаряда (`spell_projectile`):
   - Arcane Bolt: синий шар летит по параболе от мага к манекену.
   - Verdant Flame: зелёное пламя появляется на манекене снизу (translateY 40px → 0).
   - Shadow Strike: фиолетовая вспышка снизу под манекеном (scale 0 → 1.5 → 0).
4. **t=800ms**: `dummy_figure` hit-состояние:
   - Манекен: flash (opacity 1 → 0.3 → 1, 150ms) + translateX 4px → -4px → 0 (shake, 200ms).
5. **t=800ms**: `damage_number` появляется над манекеном:
   - Текст: "-[N]" (число урона с минусом).
   - Позиция старта: center-x манекена, top 20% манекена.
   - Анимация: translateY -40px за 1000ms + opacity 1 → 0 начиная с 600ms.
6. **t=900ms**: `enemy_hp_bar_fill` анимируется до нового значения (width transition 400ms).
7. **t=1300ms**: `round_counter` обновляется до следующего раунда.
8. **t=1400ms**: Кнопки → enabled (opacity 1, transition 0.3s).
   - Исключение: если HP манекена <= 0 или раунд = 3 → кнопки не возвращаются, экран переходит в transition.

### Переход к результату после 3-го раунда
- **t=1400ms от последнего удара**: начинается fade-out всего экрана (opacity → 0, 0.4s).
- **t=1800ms**: переход на BattleResultScreen (fade-in).

## Состояния экрана

### Default (начало боя)
- HP манекена: 100.
- Раунд: 1/3.
- Все кнопки активны.

### В процессе (1-2 раунд)
- HP манекена уменьшено.
- round_counter показывает текущий раунд.
- Кнопки: после выбора — disabled на время анимации, затем снова enabled.

### Конец боя (раунд 3 завершён)
- Кнопки исчезают / fade-out.
- Сцена: короткая пауза 0.5s, затем fade-out экрана.
- Данные боя (итоговый HP манекена, total damage) передаются на BattleResultScreen.

## Notes for Developer

- Состояние HP манекена хранить в локальной переменной на время боя, не в localStorage (сохранять только итог).
- `damage_number`: создавать как динамический DOM-элемент при каждом ударе, удалять после анимации (1.5s после появления).
- Расчёт урона по формуле: `floor(Random(min, max) * (1 + bonusPower/100))`.
- `bonusPower` = сумма BonusPower экипированных предметов из localStorage.
- После 3-го раунда вычислить результат: если `dummy_hp_remaining <= 0` → победа, иначе → провал.
- `fights_remaining` = `5 - combat.fightsToday`.
- Уменьшить `combat.fightsToday` при старте нового боя (не при выборе заклинания).
- `back_button` при активной анимации: использовать флаг `isAnimating = true/false`.

## Notes for Art Director

- Фон экрана: каменный тренировочный двор, ночь, каменные стены, несколько факелов по бокам. Тёмно-зелёный и синий оттенки. Трава между камнями. Референс: атмосфера как в photo_2026-03-15_16-37-25.jpg (маг у стены крепости).
- `mage_figure`: вид сзади, зелёная остроконечная шляпа, тёмно-зелёная/чёрная накидка. Размер: занимает нижнюю треть кадра. Не мультяшный — реалистичный силуэт.
- `dummy_figure`: деревянный манекен в форме T (крест-перекладина для рук). Тёмное дерево, потёртое. Не мультяшный. На среднем плане по центру.
- Кнопки заклинаний: иконки не emoji — рисованные SVG или пиксельные спрайты. Синий шар для Arcane Bolt, пламенный конус для Verdant Flame, тень-острие для Shadow Strike.
- Цифра урона: белая с тёмным обводом (1px dark outline), шрифт Cinzel bold. Хорошо читается на любом фоне сцены.
